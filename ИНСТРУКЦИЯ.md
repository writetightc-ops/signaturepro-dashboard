# SignaturePro — Инструкция по работе с системой учёта ЗП

---

## Обзор системы

Система состоит из трёх связанных инструментов:

| Инструмент | Назначение | Кто заполняет |
|---|---|---|
| **Таблица заказов** | Ведение клиентов по этапам | Менеджеры |
| **Таблица руководства** | Ставки, история ЗП, доплаты | Руководство |
| **Дашборд** | Просмотр и фиксация ЗП | Руководство |

---

## 1. ТАБЛИЦА ЗАКАЗОВ

### Структура столбцов

| Столбец | Название | Что вносить |
|---|---|---|
| **A** | Поступление | Дата прихода заказа |
| **B** | Статус | ✓ (галочка) когда заказ завершён |
| **C** | Менеджер | Имя менеджера точно как в таблице Ставки |
| **D** | Каллиграф | Имя каллиграфа точно как в таблице Ставки |
| **E** | Клиент | Имя клиента (кириллица = RU, латиница = USA) |
| **F** | Тариф | E / ST / E_FAST / OPTNEW / PRNEW / ДОП_ОБУЧЕНИЕ / ДОП_ПОДПИСЬ |
| **G** | Варианты | **Дата** сдачи вариантов (формат 11.05) |
| **H** | ОС 1 | Дата отправки обратной связи 1 |
| **I** | Правка 1 | Дата получения правки 1 |
| **J** | ОС 2 | Обратная связь 2 |
| **K** | Правка 2 | Правка 2 |
| **L–O** | ОС 3 / Правка 3 / ОС 4 / Правка 4 | Аналогично |
| **P** | Обучение | Дата проведения обучения |
| **Q** | Ссылка | Ссылка на папку с файлами |
| **R** | Завершение | Дата завершения заказа |
| **S** | Заметки | Любые комментарии |
| **T–W** | ОС 5 / Правка 5 / ОС 6 / Правка 6 | Дополнительные правки |
| **X** | Итого ЗП | **Автоформула** — не трогать! |

### Листы (вкладки)

- Каждый месяц — **отдельная вкладка**: `Апрель_2026`, `Май_2026` и т.д.
- Название вкладки должно содержать год (2026) — дашборд читает его для правильного распознавания дат.
- Все активные (незавершённые) заказы переносятся на новый лист через кнопку **«Перенести заказы»** в дашборде. **Завершёнными** считаются строки с галкой в столбце **B (Статус)** — они остаются в исходном листе и не переносятся. Переносятся все строки без галки.

### Важные правила заполнения

1. **Имена менеджера и каллиграфа** должны совпадать с именами в листе «Ставки» таблицы руководства (с учётом регистра и пробелов).
2. **Тариф** — только значения из списка: `E`, `ST`, `E_FAST`, `OPTNEW`, `PRNEW`, `ДОП_ОБУЧЕНИЕ`, `ДОП_ПОДПИСЬ`. Регистр не важен.
3. **Столбец «Варианты» (G)** — вносить **дату** (не галочку, не цифру). При клике появляется календарь.
4. **Все даты** отображаются в формате `дд.мм` (11.05 = 11 мая). Год хранится, просто не показывается.
5. **Клиент**: если имя написано кириллицей — заказ считается российским (RU), латиницей — американским (USA). Это влияет на разбивку ЗП по странам.
6. **Столбец X (Итого ЗП)** — формула считается автоматически. Если формула исчезла, запусти `setup_google_sheet.py`.
7. **Статус (B)** — ставить галочку `TRUE` когда заказ **полностью завершён**. Именно галка в B (а не дата в R) служит сигналом завершения: она определяет перенос между месяцами, начисление «бонуса без правок» и менеджерского бонуса.

### Цветовое кодирование строк

| Цвет | Значение |
|---|---|
| 🟢 Зелёный | Заказ завершён (статус = TRUE) |
| 🟡 Жёлтый | Тарифы E, ST, E_FAST |
| ⬜ Серый | Тарифы OPTNEW, PRNEW, ДОП_ОБУЧЕНИЕ, ДОП_ПОДПИСЬ |

## 2. ТАБЛИЦА РУКОВОДСТВА

Состоит из 4 листов.

---

### Лист «Ставки»

Хранит тарифные ставки и список сотрудников. **Дашборд читает ставки отсюда** (обновление раз в 10 минут).

#### Раздел «Тарифные ставки»

| Тариф | Варианты | Обучение | Бонус без правок | Тарифный бонус | Правка | Бонус менеджера |
|---|---|---|---|---|---|---|
| E | 150 | 200 | 100 | — | 75 | 200 |
| ST | 150 | 200 | 100 | — | 75 | 200 |
| E_FAST | 150 | 200 | 150 | 300 | 75 | 200 |
| OPTNEW | 300 | 400 | 200 | — | 75 | 300 |
| PRNEW | 500 | 200 | 250 | 300 | 75 | 300 |
| ДОП_ОБУЧЕНИЕ | 0 | 200 | 0 | — | 0 | 0 |
| ДОП_ПОДПИСЬ | 250 | 200 | 100 | — | 75 | 200 |

> Все значения в рублях. Изменение в этой таблице автоматически применяется в дашборде.
> **ДОП_ОБУЧЕНИЕ** — доплата за обучение: вознаграждение только в столбце «Обучение», менеджер бонус не получает.
> **ДОП_ПОДПИСЬ** — доплата за подпись (6 подписей + 2 обучения).
> **Менеджер** получает бонус один раз на клиента (при передаче клиента другому каллиграфу — две строки, но бонус менеджеру начисляется один раз).

#### Раздел «Сотрудники»

| Поле | Описание |
|---|---|
| Имя | Должно совпадать с именем в таблице заказов |
| Роль | Каллиграф / Менеджер |
| Коэффициент | 1.0 = стандарт, 1.2 = +20%, 0.8 = −20% |
| Активен | ДА / НЕТ — только «ДА» попадают в расчёт |

> Для добавления нового сотрудника — добавь строку в этот раздел и в таблицу заказов.

---

### Лист «Корректировки» (доплаты и вычеты)

Здесь вручную вносятся премии, доплаты, авансы и вычеты.

| Столбец | Описание | Ввод |
|---|---|---|
| **A — Период** | К какому периоду относится | Выпадающий список из закрытых периодов |
| **B — Сотрудник** | Кому начисление | Выпадающий список сотрудников |
| **C — Тип** | Вид начисления | Выпадающий список |
| **D — Сумма** | Сумма в рублях | Положительная = доплата, **отрицательная = вычет** |
| **E — Комментарий** | Пояснение | Свободный текст |
| **F — Дата добавления** | Когда внесено | Вручную |

**Типы:**
- **Премия** / **Доплата** — прибавляется к ЗП
- **Аванс** — фиксация выданного аванса
- **Вычет** / **Штраф** — вводи со знаком минус в Сумме

> ⚠️ Корректировки привязываются к периоду по строгому совпадению строки (например: `01.05.2026 — 31.05.2026`). Формат берётся из выпадающего списка автоматически.

---

### Лист «История_ЗП»

Архив зафиксированных выплат. **Заполняется автоматически дашбордом** через кнопку «Зафиксировать ЗП». Вручную не редактировать.

Каждая строка = один сотрудник за один период. Хранит:
- Базовую ЗП с разбивкой по компонентам (варианты, правки, обучение, бонусы)
- Коэффициент и итоговую сумму
- Разбивку по странам (USA / RU)

---

### Лист «Итого_к_выплате»

Сводный лист для бухгалтерии. **Только для чтения** — все данные формульные.

**Как пользоваться:**
1. В ячейке **C3** выбери период из выпадающего списка (или введи вручную в формате `01.05.2026 — 31.05.2026`).
2. Лист покажет:

| Строка | Данные |
|---|---|
| **Итого к выплате** | ЗП + все доплаты/вычеты за период |
| **Базовая ЗП** | Только из История_ЗП |
| **Доплаты (±)** | Сумма из Корректировок |
| **🇺🇸 USA / 🇷🇺 RU** | Разбивка по странам |

**Раздел «СВОДКА ПО СОТРУДНИКАМ»** (строки 11–27):
Автоматически показывает каждого сотрудника с его базовой ЗП, доплатами и итоговой суммой к выплате.

**Раздел «ИСТОРИЯ ЗП / ДОПЛАТЫ»** (строка 29+):
Детализация — все строки из истории и корректировок за выбранный период.

## 3. ДАШБОРД

Адрес: **https://signaturepro-dashboard.onrender.com/**

---

### Выбор периода

- Кнопки быстрого выбора: **Сегодня / Эта неделя / Этот месяц / Прошлый мес.**
- Или вручную через поля «С:» и «По:»
- Дашборд считает ЗП **за все листы** таблицы заказов одновременно

### Карточки сотрудников

- Каждая карточка = один сотрудник за выбранный период
- **ЗП скрыта по умолчанию** — нажми на карточку чтобы увидеть сумму
- После раскрытия видны прогресс-бары по компонентам и разбивка 🇺🇸/🇷🇺
- Кнопка **«Детализация»** — список всех заказов с начислениями

### Как считается ЗП каллиграфа

ЗП начисляется **по дате события** — если дата попадает в выбранный период:

| Событие | Условие начисления |
|---|---|
| **Варианты** | Дата варианты (G) в периоде |
| **Тарифный бонус** | Вместе с вариантами (только E_FAST и PRNEW) — каллиграфу, который готовил варианты |
| **Правки** | Каждая дата правки в периоде × ставка правки |
| **Обучение** | Дата обучения (P) в периоде |
| **Бонус без правок** | Галка в B + дата завершения (R) в периоде + по заказу нет ни одной правки |

> Один заказ может начислять ЗП в разных месяцах: например, варианты в апреле, правки в мае.
> Если заказ начал один каллиграф, а завершает другой — создаются **две строки на клиента**: варианты + тарифный бонус уходят первому каллиграфу (по дате в G его строки), завершающие бонусы — второму.

### Как считается ЗП менеджера

Менеджер получает бонус **при завершении заказа** — когда стоит галка в B и дата завершения (R) попадает в период. Бонус начисляется **один раз на клиента** за период: если клиента передали другому каллиграфу (две строки), менеджер получает бонус только по первой завершённой строке. По тарифу **ДОП_ОБУЧЕНИЕ** менеджер бонус не получает.

### Коэффициент

Если у сотрудника в таблице «Ставки» коэффициент ≠ 1.0, итоговая ЗП умножается на него:
> 25 000 × 1.2 = **30 000 ₽**

### Кнопка «Обновить»

Сбрасывает кеш и загружает свежие данные из Google Sheets. Использовать после внесения изменений в таблицу.

> Данные кешируются на **1 час**. Для немедленного обновления — нажми «Обновить».

### Кнопка «Перенести заказы»

Переносит **незавершённые заказы** (без галки в столбце B) из одного листа в другой. Строки с галкой в B (завершённые) остаются в исходном листе.

**Как использовать:**
1. Нажать «Перенести заказы»
2. Выбрать лист-источник (например, `Апрель_2026`)
3. Ввести или выбрать целевой лист (например, `Май_2026`)
4. Нажать «Перенести»

> ЗП продолжает считаться корректно — дашборд читает данные со всех листов.

### Кнопка «Закрыть период»

Фиксирует рассчитанную ЗП в листе «История_ЗП» таблицы руководства.

**Когда нажимать:** в конце месяца, после проверки всех начислений.

**Процесс:**
1. Выбери нужный период (например, «Прошлый мес.»)
2. Нажми «Закрыть период»
3. Проверь суммы в открывшемся модальном окне
4. Нажми «Зафиксировать ЗП» — подождать 5–10 секунд
5. Появится зелёная надпись — период зафиксирован
6. Рядом с периодом появится бейдж «Зафиксирован»

> ⚠️ Повторная фиксация одного и того же периода невозможна. Если нужно пересчитать — напиши разработчику для удаления строк из «История_ЗП».

**После фиксации:**
- Зайди в «Корректировки» и добавь премии/вычеты при необходимости
- Проверь итоговые суммы в листе «Итого_к_выплате»

## 4. ТИПИЧНЫЙ РАБОЧИЙ ПРОЦЕСС

### Ежедневно (менеджеры)
- Открыть таблицу заказов → текущий месяц
- Внести даты этапов по клиентам (варианты, ОС, правки, обучение)
- При завершении заказа: поставить галочку в B + дату в R

### Конец месяца (руководство)
1. Открыть дашборд → выбрать «Прошлый мес.»
2. Нажать **«Обновить»** для получения свежих данных
3. Проверить суммы по каждому сотруднику (раскрыть карточки)
4. Нажать **«Закрыть период»** → **«Зафиксировать ЗП»**
5. В таблице руководства → лист **«Корректировки»** — добавить премии/вычеты
6. Открыть лист **«Итого_к_выплате»** → выбрать период → проверить финальные суммы
7. В начале нового месяца: в дашборде нажать **«Перенести заказы»** → перенести незавершённые на новый лист

### Перенос заказов на новый месяц
1. Убедиться что все завершённые заказы апреля отмечены галкой в столбце **B** (и проставлена дата в R)
2. В дашборде → **«Перенести заказы»**
3. Из: `Апрель_2026` → В: `Май_2026`
4. Нажать «Перенести» — строки без галки в B переедут, с галкой останутся; форматирование сохранится

## 5. ЧАСТО ЗАДАВАЕМЫЕ ВОПРОСЫ

**❓ Почему ЗП в дашборде отличается от столбца X в таблице?**
Столбец X показывает общую сумму за весь заказ (без фильтра по периоду). Дашборд считает только по датам, попадающим в выбранный период.

**❓ Почему сотрудник не отображается в дашборде?**
Проверить: 1) имя в таблице заказов совпадает с именем в «Ставках» (пробелы, регистр); 2) в «Ставках» у сотрудника стоит «ДА» в столбце «Активен».

**❓ Данные устарели — как обновить?**
Нажать кнопку **«Обновить»** в дашборде. Данные кешируются на 1 час.

**❓ Как изменить ставки?**
Открыть таблицу руководства → лист «Ставки» → изменить значение → дашборд подхватит изменения в течение 10 минут (или сразу после нажатия «Обновить»).

**❓ Как добавить нового сотрудника?**
1. В таблице руководства → лист «Ставки» → добавить строку в раздел «Сотрудники»
2. В таблице заказов писать его имя точно как в «Ставках»

**❓ Как добавить премию?**
В таблице руководства → лист «Корректировки» → добавить строку: выбрать период, сотрудника, тип «Премия», ввести сумму.

**❓ Период закрыт, но нужно пересчитать ЗП**
Написать разработчику — строки из «История_ЗП» для данного периода будут удалены, после чего можно будет зафиксировать заново.
