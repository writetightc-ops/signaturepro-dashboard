SignaturePro — Публикация дашборда

Пошаговая инструкция: от кода до постоянной публичной ссылки

Нажимай на каждый шаг, чтобы развернуть инструкцию. Шагов всего 4.

1
Установи GitHub Desktop
5 мин · легко ▼

GitHub Desktop — это программа с красивым интерфейсом для загрузки кода в облако. Командная строка не нужна.

Скачать GitHub Desktop
  1. Скачай и установи GitHub Desktop
  2. При запуске нажми «Sign in to GitHub.com»
  3. Создай аккаунт или войди (можно через Google)
  4. После входа увидишь главный экран — готово
2
Загрузи код на GitHub
3 мин · легко ▼

Загружаем папку с проектом на GitHub. Файлы с данными и ключами не попадут в облако (они в .gitignore).

  1. В GitHub Desktop: File → Add local repository
  2. В поле «Local path» укажи папку:
    d:\Подписи курсор
  3. Если появится сообщение «This directory does not appear to be a Git repository» — нажми «create a repository»
  4. В поле «Name» напиши: signaturepro-dashboard
  5. Нажми «Create repository»
  6. Нажми синюю кнопку «Publish repository» вверху
  7. Убери галочку «Keep this code private»
  8. Нажми «Publish repository»
Репозиторий появится на github.com — в нём будут только файлы кода (.py, .html), без Excel и credentials.json
3
Создай сервис на Render
10 мин ▼

Render — бесплатный хостинг для Python-приложений. Подключается к GitHub и запускает твой код автоматически.

Открыть render.com
  1. Нажми «Get Started for Free»
  2. Войди через «GitHub» — это важно, чтобы Render видел твои репозитории
  3. После входа нажми «New +» → «Web Service»
  4. Выбери репозиторий signaturepro-dashboard → нажми «Connect»

На странице настройки сервиса проверь:

ПолеЗначение
Namesignaturepro-dashboard
RuntimePython 3
Build Commandpip install -r requirements.txt
Start Commandgunicorn --workers 1 --timeout 120 dashboard:app
Instance TypeFree
Если Start Command уже заполнен автоматически из Procfile — всё правильно, не меняй.
4
Добавь переменные окружения и запусти
5 мин ▼

Перед тем как нажать «Create», прокрути вниз до раздела «Environment Variables» и добавь две переменные:

Переменная 1 — URL Google Таблицы:

KeyValue
GOOGLE_SHEETS_URL Скопируй URL своей Google Таблицы из браузера

Переменная 2 — Ключ доступа к Google Sheets:

KeyValue
GOOGLE_CREDENTIALS_B64 Содержимое файла RENDER_CREDENTIALS_B64.txt из папки проекта
Файл с ключом: d:\Подписи курсор\RENDER_CREDENTIALS_B64.txt
Открой его в блокноте (ПКМ → «Открыть с помощью» → Блокнот), выдели всё (Ctrl+A) и скопируй (Ctrl+C)
Файл RENDER_CREDENTIALS_B64.txt содержит секретный ключ — никому не отправляй его содержимое. После настройки Render можно удалить этот файл с компьютера.
  1. После добавления обеих переменных нажми «Create Web Service»
  2. Render начнёт сборку — займёт 2–4 минуты
  3. Когда статус станет «Live» — дашборд доступен!
Ссылка появится вверху страницы Render — вида https://signaturepro-dashboard.onrender.com
Это и есть постоянный публичный адрес. Поделись им с командой.

Частые вопросы
FAQ ▼

Как обновить код после изменений?

В GitHub Desktop нажми «Commit to main» → «Push origin». Render автоматически подхватит изменения и перезапустит сервер.

Что если Render «засыпает»?

На бесплатном тарифе сервис засыпает после 15 мин бездействия. Первый запрос будет ждать ~30 секунд (пробуждение). Это нормально. Чтобы этого не было — нужен платный тариф ($7/мес).

Как запускать локально (для офлайн работы)?

cd "d:\Подписи курсор" python dashboard.py

Открыть: http://localhost:5000

Как обновить данные в дашборде?

Нажми зелёную кнопку «Обновить» в шапке дашборда. Данные загрузятся из Google Sheets.

Как перенести незавершённые заказы на новый месяц?

Нажми кнопку «Перенести заказы» в шапке → выбери исходный лист → введи название нового (например Май_2026) → нажми «Перенести».

Готово! Дашборд доступен всей команде

Постоянная ссылка на render.com — не нужно держать компьютер включённым.
Данные всегда актуальны из Google Sheets.
